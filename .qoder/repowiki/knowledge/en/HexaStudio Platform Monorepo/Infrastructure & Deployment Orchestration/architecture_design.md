Two cooperating sub-trees under a single `docker compose` project:

- `docker/<service>/` ā€” per-service configuration trees consumed by Docker Compose. Each service is self-contained: `traefik/` (entrypoint + ACME TLS via `traefik.yml`, dynamic hostā†’service routing in `dynamic.yml`), `nginx/` (legacy proxy config for local dev), `odoo/` (Odoo ERP with addons mount), `postgres/init/01-databases.sql` (bootstraps `hexastudio_api` and `hexastudio_cms`), `prometheus/prometheus.yml` (scrapes backend, traefik, node-exporter), `loki/` (`loki-config.yml` filesystem-backed TSDB + `promtail-config.yml` Docker log scraper), `grafana/provisioning/datasources/datasources.yml` (provisions Prometheus/Loki as default data sources), `minio/init-buckets.sh` (waits for MinIO then creates private buckets), `backup/backup.sh` (cron loop that `pg_dump`s both databases to `/backups` and optionally uploads to MinIO).
- `scripts/` ā€” operational tooling. `deploy.sh` is the CLI entry point implementing zero-downtime rolling updates (pull ā†’ `up -d --no-deps <svc>` ā†’ health-check loop ā†’ rollback on failure) plus helpers (`db-migrate`, `db-seed`, `shell`, `clean`). `setup-server.sh` hardens a fresh Ubuntu 24.04 host (fail2ban, UFW, SSH lockdown, unattended-upgrades). `generate-secrets.sh` copies `.env.example` into `.env.production` and fills every secret placeholder with `openssl rand`. `scripts/mcp/` is a standalone Node.js MCP server (`server.js` over SSE at `/sse`) exposing tools like `run-command`, `docker-compose`, `read-file`, etc., built via `scripts/Dockerfile.mcp`.

Dependency direction: `deploy.sh` orchestrates `docker compose`; `backup.sh` depends on `postgres` and `minio` services; `promtail` scrapes Docker logs and pushes to `loki`; `grafana` consumes `prometheus` and `loki`; Traefik routes external domains to internal containers. There is no shared library ā€” each file is a leaf artifact mounted into its container.
