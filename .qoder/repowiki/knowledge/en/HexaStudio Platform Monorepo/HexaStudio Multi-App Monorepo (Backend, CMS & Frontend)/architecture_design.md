Three sibling apps under `apps/`, each a self-contained Node package with its own `Dockerfile` and build pipeline:

- `backend/` — NestJS application (`src/main.ts` bootstrap) wired via `src/app.module.ts`. A global `ValidationPipe`, `GlobalExceptionFilter`, Helmet, Sentry, Swagger (dev-only), CORS, and `ThrottlerGuard` are registered at the app root; feature modules live under `src/modules/<domain>/` (auth+jwt guard+strategy, projects, articles, services, contact, storage via MinIO + Redis, Odoo webhook, portal, users, email, requests, accounting). Cross-app shared types come from workspace packages `@hexastudio/types`, `@hexastudio/utils`.
- `cms/` — Strapi v5 project (`@strapi/strapi`) exposing custom APIs for article, category, portfolio, and service content-types under `src/api/<type>/` (schema.json + controller + route + service). Uses PostgreSQL (`pg`) and ships with `@strapi/plugin-users-permissions`.
- `frontend/` — Next.js 15 app (standalone output) using React Server Components plus client components in `src/components/`, domain features under `src/features/<domain>/` (blog, portfolio, scene, auth, services), Zustand stores, TanStack Query, Three.js / @react-three/fiber/drei for the interactive studio experience, Tailwind CSS v4, Framer Motion, GSAP, Lenis smooth scroll, and Sentry SDKs for client/edge/server.

Dependency direction: frontend → backend REST API + Strapi admin API; cms is independent and only consumed by the frontend. All three apps are orchestrated by Turborebo (`turbo-*` logs present) and share workspace packages referenced via `*` version in their `package.json`.
