Top-level npm workspaces + Turbo orchestrate five packages: `apps/api` (NestJS REST API with TypeORM/PostgreSQL, global `/api` prefix, ValidationPipe whitelist+transform), `apps/realtime` (standalone Socket.IO server with Redis pub/sub adapter for horizontal scaling and JWT auth middleware joining per-user rooms), `apps/worker` (BullMQ consumers on email/notifications/ai queues with graceful SIGTERM/SIGINT shutdown), `apps/web` (Next.js frontend consuming the API and realtime service), and `packages/types` (shared DTO/entity type definitions re-exported from `index.ts`). The API is organized by domain (`modules/{auth,users,workspaces,projects,messages,ai,odoo}`) each owning its controller/service/module/entities/dto; cross-cutting concerns live under `common`, `config`, and `database`. All four apps are containerized via per-app Dockerfiles and orchestrated together with `docker/docker-compose.yml` which provisions PostgreSQL, Redis, MinIO, and the three Node services. Dependency direction is one-way: `web` → `api` + `realtime`; `worker` → Redis only; `packages/types` is consumed by both `api` and `worker`.
