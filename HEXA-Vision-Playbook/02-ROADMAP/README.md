# Sprints Directory

This directory contains sprint plans, backlogs, and retrospectives.

## Contents

| File | Description |
|------|-------------|
| `backlog.md` | Product backlog (all unplanned items) |
| `current.md` | Current active sprint |
| `sprint-001.md` | Sprint 1 plan |
| `sprint-001-review.md` | Sprint 1 retrospective |

## Structure

Each sprint document should include:

1. **Sprint Goal** (one sentence)
2. **Sprint Dates** (start/end)
3. **Team Capacity** (story points or hours)
4. **Backlog Items** (with owner, status, priority)
5. **Risks** (identified blockers)
6. **Definition of Done** (sprint-specific)

## Sprint Retrospective Format

1. **What went well**
2. **What could be improved**
3. **Action items**
4. **Team health check**
