# 📖 DEFINITIONS: THE UNIFIED LANGUAGE

**Version:** 1.0 | **Domain:** Governance | **Purpose:** Semantic Consistency

## 1. THE IMPORTANCE OF SEMANTICS
In a multidisciplinary team (Architects, Designers, Engineers), a single word can mean three different things. To avoid catastrophic misunderstandings, we define our terms strictly.

---

## 2. CORE DOMAIN DEFINITIONS

### I. The "Experience" (UX/UI)
- **Cinematic Flow:** A sequence of views that feels like a movie rather than a website.
- **Invisible Navigation:** UI that is available when needed but does not distract from the content.
- **Micro-Interaction:** A small, functional animation that provides tactile feedback (e.g., a button "pressing" down).
- **Luxury Gap:** The perceived difference between a "standard" professional site and the HEXA experience.

### II. The "Engine" (Technical)
- **BFF (Backend-for-Frontend):** A specialized backend layer (NestJS) that aggregates data from multiple sources (Strapi, DBs) and optimizes it for a specific frontend.
- **SOT (Single Source of Truth):** The authoritative source of a piece of data. In this project, `hexa-platform` is the SOT.
- **LOD (Level of Detail):** A technique where the complexity of a 3D model is reduced as it moves further from the camera.
- **Draco Compression:** A library for compressing 3D geometric meshes and point clouds.

### III. The "Process" (Governance)
- **ADR (Architecture Decision Record):** A document that captures a technical choice, the alternatives considered, and the rationale.
- **Gold Standard:** The highest possible quality level, defined in the Playbook, against which all work is measured.
- **Quality Gate:** A mandatory review process that must be passed before code is merged.

---

## 3. PROJECT SPECIFIC TERMS
- **Project Manifest:** A JSON file containing all the assets, coordinates, and metadata required to render a specific 3D architectural project.
- **The "Void":** The deep black background used in the design system to create high contrast for 3D models.
- **Symmetry Law:** The requirement that all frontend components be perfectly synced with backend types.

*“Precision in language leads to precision in execution.”*
