# 📚 GLOSSARY: THE HEXA LEXICON

**Version:** 1.0 | **Domain:** Governance | **Format:** A-Z Reference

## A
- **Awwwards:** A professional web design competition. We use their "Site of the Day" as our visual benchmark.
- **BFF:** Backend-for-Frontend. A design pattern used to optimize the API for the specific needs of the frontend.

## B
- **Baking (Lighting):** The process of pre-calculating light and shadows into a texture map to save GPU resources at runtime.

## C
T-**CLS:** Cumulative Layout Shift. A metric that measures the visual stability of a page.

## D
- **Draco:** A library for compressing and decompressing 3D meshes.
- **DTO:** Data Transfer Object. An object that defines how data is sent over the network.

## E
- **Enterprise-Grade:** Software that is scalable, secure, maintainable, and documented.

## F
- **FPS:** Frames Per Second. Our target is a stable 60 FPS for all 3D interactions.

## G
- **GSAP:** GreenSock Animation Platform. Our primary tool for high-performance, cinematic timelines.

## H
- **HDR (High Dynamic Range):** Imaging that allows for a greater range of luminosity, used for realistic 3D environments.

## I
- **ISR:** Incremental Static Regeneration. A Next.js feature that allows us to update static pages without a full rebuild.

## L
- **LCP:** Largest Contentful Paint. The time it takes for the largest element on the page to become visible.

## M
- **Monorepo:** A single repository containing multiple projects (frontend, backend, types).

## N
- **Next.js 15:** Our frontend framework, chosen for its speed, SEO, and App Router capabilities.

## O
- **Odoo:** The ERP system used for business operations and as a content source for some modules.

## P
- **PostgreSQL:** Our primary relational database for structured data.

## R
- **R3F (React Three Fiber):** A React renderer for Three.js.

## S
- **Strapi 5:** Our headless CMS used for managing architectural project content.

## T
- **Three.js:** The core JavaScript library used to render 3D graphics in the browser.
- **TypeScript:** Our primary language, used to ensure type-safety and reduce runtime errors.

## Z
- **Zustand:** A lightweight state management library used for client-side application state.

*“A shared vocabulary is the foundation of a shared vision.”*
