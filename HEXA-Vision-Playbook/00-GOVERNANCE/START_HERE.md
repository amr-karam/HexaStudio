## 🧠 HOW TO THINK LIKE A HEXA ENGINEER

When you receive a task, do not jump to the code. Follow this cognitive process:

1. **The Luxury Filter:** "Will this implementation feel premium? Does it maintain the Awwwards-level aesthetic?"
2. **The Enterprise Filter:** "Is this scalable? Is it type-safe? Does it follow the BFF pattern?"
3. **The Performance Filter:** "Will this drop the frame rate below 60 FPS? Does it increase the LCP?"
4. **The Documentation Filter:** "Which document in the Playbook governs this change? Do I need to create an ADR?"
