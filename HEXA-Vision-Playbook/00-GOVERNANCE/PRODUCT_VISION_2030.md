## 10-Year Strategic Outlook

This document defines the long-term vision for HEXA Vision. It is aspirational but grounded. Every decision today should move us closer to this future.

---

## Vision 2030 Statement

> **HEXA Vision will be the global operating system for architecture visualization firms — powering every aspect of their business from first client meeting to final delivery, with AI-augmented 3D experiences that redefine how architecture is presented and experienced.**

---

## Market Thesis

### Current State (2026)

- Architecture visualization is dominated by static imagery
- Most firms use 5-10 disconnected tools (website, email, Dropbox, Excel, etc.)
- AI in architecture is nascent, mostly experimental
- Client experience is fragmented (email proposals, PDF portfolios, in-person reviews)
- 3D in the browser is technically feasible but poorly executed by most

### 2030 Prediction

- **AI-generated 3D content** will be the norm, not the exception
- **Real-time collaborative review** will replace email-based feedback
- **VR/AR** will be standard for architectural presentations
- **Automated business operations** will be table stakes for competitive firms
- **Data-driven design** will connect project metrics to business outcomes

## Strategic Pillars for 2030

### Pillar 1: Immersive Intelligence

**Today:** Static 3D scenes with manual navigation
**2030:** AI-generated, responsive 3D environments that adapt to viewer preferences

| Capability | 2026 | 2028 | 2030 |
|------------|------|------|------|
| 3D rendering | WebGL | WebGPU | Native-quality ray tracing in browser |
| AI scenes | Manual creation | AI-assisted layout | Fully AI-generated environments |
| Interaction | Mouse/keyboard | Touch + voice | Gesture + eye tracking + voice |
| Personalization | None | Viewing preferences | AI-curated experiences per viewer |
| Multiplayer | None | Shared viewing | Real-time collaborative walkthroughs |

### Pillar 2: Autonomous Business Platform

**Today:** Automated workflows with manual oversight
**2030:** Self-operating business with AI decision-making

| Capability | 2026 | 2028 | 2030 |
|------------|------|------|------|
| Lead management | Automated capture | AI qualification | Predictive lead scoring + auto-response |
| Project management | Semi-automated | Most workflows automated | Fully autonomous project orchestration |
| Document generation | Template-based | AI-assisted drafting | Auto-generated with AI review |
| Resource planning | Manual | AI-suggested | Fully optimized auto-scheduling |
| Invoicing | Auto-generated | Smart payment reminders | Predictive cash flow management |

### Pillar 3: Predictive Analytics

**Today:** Descriptive analytics (what happened)
**2030:** Prescriptive analytics (what to do next)

| Capability | 2026 | 2028 | 2030 |
|------------|------|------|------|
| Analytics | Basic metrics | Advanced dashboards | AI-driven insights |
| Reporting | Manual exports | Automated reports | Natural language query interface |
| Prediction | None | Trend identification | AI-powered business forecasting |
| Optimization | None | Suggested improvements | Auto-optimization of resources |

### Pillar 4: Universal Client Experience

**Today:** Web portal with basic features
**2030:** Immersive client collaboration platform

| Capability | 2026 | 2028 | 2030 |
|------------|------|------|------|
| Client portal | Desktop web | Mobile + web | VR/AR + web + mobile |
| Review process | File-based comments | In-scene markup | Real-time co-design |
| Communication | Email + portal | In-app chat + video | AI-mediated async communication |
| Feedback | Manual forms | Structured input | Sentiment analysis + auto-action |
| Delivery | Download links | Streaming | Interactive 3D delivery |

### Pillar 5: Ecosystem & Marketplace

**Today:** Closed platform
**2030:** Open ecosystem with third-party extensions

| Capability | 2026 | 2028 | 2030 |
|------------|------|------|------|
| Integrations | Odoo, Strapi | API marketplace | Open plugin ecosystem |
| Templates | Internal | Curated library | Community marketplace |
| 3D assets | Internal models | Partner library | Peer-to-peer asset exchange |
| Extensions | None | Custom modules | Third-party app store |

## Technology Roadmap

### 2026-2027 (Foundation)

- Next.js 15 with App Router
- React Three Fiber / WebGL
- TailwindCSS design system
- NestJS BFF layer
- Odoo ERP integration
- PostgreSQL, Redis, MinIO
- Docker Compose orchestration
- GitHub Actions CI/CD

### 2028-2029 (Evolution)

- WebGPU for enhanced 3D rendering
- Real-time collaboration (WebSockets/WebRTC)
- AI content generation (text-to-3D, text-to-image)
- Mobile apps (React Native)
- VR/AR support (WebXR)
- Multi-region deployment
- Event-driven architecture (Kafka/RabbitMQ)

### 2030+ (Revolution)

- Native-quality ray tracing in browser
- Fully AI-generated 3D environments
- Gesture and voice interaction
- Predictive business automation
- Open plugin ecosystem
- Decentralized asset marketplace
- AR glasses / spatial computing support

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| WebGPU adoption slower than expected | Delays in 3D roadmap | Maintain WebGL fallback |
| AI costs prohibitive for target market | Pricing pressure | Tiered AI plans, on-device inference |
| Odoo limitations for scale | Architecture constraints | Abstract Odoo behind BFF, swap if needed |
| Browser fragmentation | Support burden | Progressive enhancement, graceful degradation |
| Talent availability | Development velocity | AI-augmented development, comprehensive docs |
