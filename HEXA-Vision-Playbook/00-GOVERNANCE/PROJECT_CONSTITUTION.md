# 🏛️ PROJECT CONSTITUTION: THE HEXA VISION STANDARD
**Version:** 1.0 | **Status:** Binding | **Authority:** Project Lead / Chief Architect

## 1. THE MANIFESTO OF EXCELLENCE
HEXA Vision is not a "website" or a "portfolio." It is a **Digital Architectural Experience**. We do not compete with other agencies; we compete with the limits of the web browser.

**Our Core Mandate:** To synthesize extreme visual fidelity with uncompromising enterprise stability. If a feature is "functional" but not "luxurious," it is a failure. If it is "beautiful" but "slow," it is a failure.

---

## 2. THE THREE PILLARS OF HEXA

### I. Visual Absolute (The Awwwards Standard)
- **Zero Compromise:** Every transition, every pixel, and every 3D interaction must feel fluid, intentional, and premium.
- **Motion as Language:** Animation is not decoration; it is communication. We use GSAP and Framer Motion to guide the user's eye and evoke emotion.
- **The Luxury Gap:** We strive for the "Luxury Gap"—the difference between a standard site and a HEXA experience. This gap is filled by attention to detail (micro-interactions, typography, and lighting).

### II. Engineering Rigor (The Enterprise Standard)
- **Stateless & Scalable:** The backend must be a fortress. We utilize NestJS and PostgreSQL with a strict BFF (Backend-for-Frontend) pattern to ensure the frontend remains lean.
- **Type-Safety as Law:** TypeScript is not a suggestion; it is a requirement. `any` is forbidden. Interfaces must be shared across the monorepo to ensure a single source of truth.
- **Performance as a Feature:** LCP < 1.2s is not a goal; it is a requirement. We optimize every asset, every query, and every render loop.

### III. Architectural Integrity (The Sustainable Standard)
- **Future-Proofing:** We build for 2030. Every design decision must be extensible.
- **Documented Truth:** If it isn't in the Playbook, it doesn't exist. Every architectural pivot must be captured in an ADR (Architecture Decision Record).
- **Maintainability > Speed:** We prefer a clean, scalable solution over a "quick fix." Technical debt is a high-interest loan that we refuse to take.

---

## 3. THE NON-NEGOTIABLES (THE "GOLDEN RULES")

1. **The 9/10 Rule:** No UI component is merged unless it scores at least 9/10 in the Luxury Feeling and Performance categories.
2. **The "Symmetry" Principle:** Frontend and Backend must be perfectly synced via shared types.
3. **The Zero-Error Policy:** Production code must have zero console errors and zero memory leaks in the 3D scene.
4. **Documentation First:** Architecture is designed and documented *before* a single line of code is written.

---

## 4. GOVERNANCE & ENFORCEMENT

This Constitution is the highest authority in the codebase. In the event of a conflict between a task request and this document, **the Constitution wins.**

**Quality Gate Controllers (QGC)** are empowered to reject any PR that violates these pillars, regardless of the feature's functionality.

*“Precision in the details, excellence in the execution.”*
